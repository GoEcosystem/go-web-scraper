Getting Started

Introduction

The Go Web Scraper is a comprehensive web scraping solution built in Go with SQLite database integration and a web interface. This guide will help you get started with using the application.

Key Features

  • Concurrent scraping of multiple websites
  • SQLite database for persistent storage
  • Web interface for viewing scraped data
  • Export to JSON or CSV formats
  • Ethical scraping with rate limiting and user agent rotation
Quick Start Commands
git clone https://github.com/GoEcosystem/go-web-scraper.git
cd go-web-scraper
go build -o scraper ./cmd/scraper
go run ./cmd/webserver -port 8080
Server running at http://127.0.0.1:8080/
Using the Web Interface

The dashboard provides an overview of your scraped data and quick access to all features:

  • Database Statistics: Shows the count of articles and products in the database
  • Quick Actions: Shortcuts to export, import, settings, and documentation
  • Available Scrapers: Run scrapers directly from the dashboard

The articles page displays all scraped articles from sources like Hacker News:

  • View article titles, authors, points, and comment counts
  • Click "Read Article" to open the original article in a new tab
  • Use pagination controls to navigate through large sets of articles

The products page displays all scraped products from e-commerce websites:

  • View product images, titles, descriptions, and prices
  • Products are organized by category with visual indicators
  • Use pagination to browse through all available products

The export page allows you to save scraped data to external files:

  • Export articles or products independently
  • Choose between JSON or CSV format
  • View and download recent exports

The import page lets you add data from external files:

  • Upload JSON or CSV files containing articles or products
  • Option to overwrite existing entries with the same identifiers
  • Data format specifications are provided on the page

The settings page provides configuration options for the scraper:

  • General settings like database path and output directory
  • Scraper settings including request delay and user agents
  • Data retention policies
  • Database management tools
Command Line Usage

The Go Web Scraper can also be used from the command line:

Basic Command Structure
./scraper -source [source_name] -output [output_file]
Available Sources
  • hackernews - Scrape articles from Hacker News
  • bookstore - Scrape products from the bookstore website
Common Options
Option Description Example
-source Specify which website to scrape -source hackernews
-output Output file path (JSON or CSV) -output data/articles.json
-format Output format (json or csv) -format csv
-limit Maximum number of items to scrape -limit 50
-db Database path for storing results -db data/scraper.db
Example Commands
./scraper -source hackernews -limit 30 -output data/hn_articles.json
./scraper -source bookstore -format csv -output data/books.csv
./scraper -source hackernews -db data/scraper.db
Ethical Scraping Guidelines

The Go Web Scraper emphasizes ethical web scraping practices:

Best Practices
  • Respect robots.txt directives
  • Implement reasonable rate limiting
  • Rotate user agents to distribute load
  • Use conditional requests to minimize bandwidth
  • Cache results when appropriate
Actions to Avoid
  • Scraping at high frequencies
  • Ignoring robots.txt directives
  • Scraping protected/private content
  • Misrepresenting yourself (e.g., fake user agents)
  • Republishing copyrighted content without permission
